Overview

Executive Summary

End-to-end migration from Shopify to a static-site architecture hosted on Cloudflare, with Decap CMS for in-house content editing.

The scope assumes 22 static HTML pages have already been designed and will be provided by the client. Our work is strictly integration and CMS setup, not design or frontend bug-fixing.

Phase 1 · Day 1

Asset Preparation & CSS Consolidation

  • Receive and audit all 22 HTML files.
  • Extract shared CSS (navigation, footer, typography, buttons, design tokens) into a common layout/template.
  • Leave page-specific CSS scoped to each template.
CSS re-adaptation is included in this quote.

Phase 2 · Day 1-2

Decap CMS Architecture & Configuration

Setup of 6 CMS collections:

  • Home (Singleton) - covers /
  • Pages (Singletons) - covers /pages/frontpage, /pages/contact-us, /pages/brands
  • Legal (Singletons) - covers /pages/privacy-policy, /pages/terms-and-conditions
  • Brands (Folder) - covers 16 brand pages including Alquema, Brax, Foil, Fragonard, Hanro, Ivko, Liverpool, Yerse, Zaket & Plover, Joseph Ribkoff, NYDJ, Raffaello Rossi, Regn, Salus, Vassalli, Velvet
  • Settings (Singleton) - global data: phone, address, hours, social links
  • Blog (Folder, add-on) - covers /blogs/news/*, /blogs/latest-news/*
Assumption: all 16 brand pages share the exact same HTML structure. If any brand page deviates significantly, re-scoping may be required.

Phase 3 · Day 2-3

Template Development

  • Build page templates that consume Decap-generated Markdown/YAML.
  • Ensure templates handle empty fields gracefully without breaking the layout.
  • Wire global settings (contact info, hours) into footer and contact sections.
  • Integrate provided meta titles and descriptions from current Shopify SEO work.

Phase 4 · Day 4

Deploy & DNS Cutover

  • Deploy to Cloudflare Pages with a temporary .pages.dev domain.
  • Run mobile, link, form, and speed tests.
  • Configure DNS cutover for empressmelbourne.com.au.
  • Submit new sitemap to Google Search Console.
  • Set up 301 redirects from old Shopify URLs.
Google Maps iframe will remain hardcoded unless otherwise requested.

Phase 5 · Post-launch

Stabilisation

  • Monitor 404s and search performance for 2-4 weeks.
  • One round of minor CMS tweaks if needed.
Shopify cancellation (client side) only after 100% confirmation that the new site is live and stable. We don't need Shopify access since audits are through Google Search Console.
In Scope

What Is Included

  • Decap CMS setup with 6 collections (5 core + 1 optional Blog add-on).
  • CSS consolidation into shared layout + page-specific scoping.
  • Static site generation pipeline (11ty / Astro / equivalent).
  • Cloudflare Pages deployment + DNS configuration.
  • 301 redirect mapping from old Shopify URLs.
  • Google Search Console re-verification + sitemap submission.
  • Mobile responsiveness testing within the constraints of the provided HTML.
  • Empty-field fail-safes in all templates.
Out of Scope

What Is NOT Included

  • Frontend bug fixes / responsive fixes in the provided HTML - can be quoted separately if issues are found.
  • Image CDN / optimisation service (Cloudinary, imgix, etc.) - images must be uploaded at correct size via CMS.
  • Blog setup - available as an add-on.
  • Google Maps iframe editing - remains hardcoded unless requested.
  • Design changes or new page creation.
  • Copywriting or SEO strategy - existing meta data will be ported as-is.
  • Advanced image cropping / transforms - aspect ratios handled by frontend CSS only.
Decap CMS outputs Markdown and frontmatter. If raw HTML is pasted into rich-text fields, it may break templates. Content must be entered as plain text or Markdown. We will enforce this via field configuration.
Assumptions

What We Assume

  • All 22 pages follow the same structural patterns demonstrated in the sample files (homepage split-hero, brand page split-hero, full-bleed secondary pages).
  • All 16 brand pages share an identical HTML structure.
  • Images will be provided at correct dimensions; the CMS will not perform server-side resizing.
  • The client will provide the complete set of 22 HTML files, plus any new images/assets, at project kickoff.
  • Domain DNS management is accessible and can be pointed to Cloudflare.
